## السياق

لفهم Azure Active Directory بشكل أفضل قررت إنشاء مستأجر (Tenant) خاص وإعداد بيئة اختبار (Lab).

سمح لي ذلك باستكشاف مزايا Azure AD مثل: إدارة المستخدمين، المزامنة مع Active Directory محلي، والتكامل مع تطبيقات سحابية.

هذه الصفحة تفصّل الخطوات التي اتبعتها لإعداد مستأجر Azure AD ومزامنة AD محلي مع السحابة.

## إنشاء مستأجر Azure

أولاً تحتاج إنشاء حساب [Microsoft Azure](https://azure.microsoft.com/en-us/pricing/purchase-options/azure-account) (يتوفر خيار مجاني تماماً).

ثم انتقل إلى البوابة: [https://portal.azure.com](https://portal.azure.com) لإنشاء مستأجر Azure AD جديد.

## إنشاء أول حساب داخل المستأجر

بعد الدخول إلى المستأجر الجديد (إن لم تفعل مسبقاً) أنشئ مستخدماً جديداً. سيُستخدم هذا الحساب لمزامنة AD المحلي مع سحابة Azure AD.

![Image 5](/images/azure/5.PNG)  
![Image 6](/images/azure/6.PNG)  
![Image 7](/images/azure/7.PNG)  

بعد إنشاء المستخدم (يستغرق 1–2 دقيقة)، انسخ UPN الخاص به. مثال: `apes@labapes.onmicrosoft.com`.

## خدمة مزامنة Azure AD Connect

بعد ضبط AAD Connect شغّل برنامج "Synchronization Service" لفحص سجلات المزامنة بين AD المحلي و Azure AD.

![Image 28](/images/azure/28.PNG)  

يجب أن ترى عمليات "Success". يمكنك أيضاً التحقق في بوابة Azure من أن المستخدمين تمت مزامنتهم.

![Image 29](/images/azure/29.PNG)  
![Image 30](/images/azure/30.PNG)  

## إنشاء حساب في AD المحلي

بعد هذه التحققات يمكنك اختبار إنشاء مستخدم جديد في AD المحلي وفرض مزامنة باستخدام الأمر:
`Start-ADSyncSyncCycle -PolicyType Delta`

![Image 31](/images/azure/31.PNG)  
![Image 32](/images/azure/32.PNG)  

كل شيء يعمل وظهر المستخدم الجديد!
